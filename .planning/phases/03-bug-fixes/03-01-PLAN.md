<objective>
Add silent close-polygon duplicate-corner removal to the Pydantic schema so legacy mask.json files (produced by matplotlib labeler's double-click auto-close) are silently cleaned during ingestion.

Purpose: Legacy mask.json files in Supabase have the last corner duplicating the first corner. This produces zero-length edges that can confuse downstream geometry. The dedup must be silent (no user-visible warning) and must protect both the CLI path (polygons_from_clicks) and the future FastAPI HTTP path (same schema).

Output: Updated schema.py with dedup validator, updated test_schema.py with dedup test coverage.

    The validator logic (per D-02, close-polygon dedup only):
    ```python
    @field_validator("corners_pix")
    @classmethod
    def strip_close_polygon_duplicate(cls, v: list[list[float]]) -> list[list[float]]:
        """Strip duplicate last corner if it matches the first (D-01, D-02, D-03).

        The matplotlib labeler's double-click auto-close produces mask.json files
        where the last corner duplicates the first. This silently removes it.
        Only strips the LAST corner, and only if it matches the FIRST within
        a small pixel-space tolerance. Does NOT strip all consecutive duplicates.
        """
        if len(v) < 2:
            return v
        first = v[0]
        last = v[-1]
        dist_sq = sum((a - b) ** 2 for a, b in zip(first, last))
        if dist_sq < 0.5 ** 2:  # 0.5 pixel tolerance
            import logging
            logging.getLogger(__name__).debug(
                "stripped duplicate close-polygon corner (dist=%.4f px)",
                dist_sq ** 0.5,
            )
            return v[:-1]
        return v
    ```

    Key details:
    - Tolerance: 0.5 pixel Euclidean distance (0.25 squared). This is pixel-space comparison per Claude's Discretion.
    - The import of logging is inside the validator to keep it self-contained. Alternatively, use the module-level log already present (add `import logging` and `log = logging.getLogger(__name__)` at module top if not already there).
    - D-03: Log at DEBUG only, no warning. Normal runs (INFO level) see nothing.
    - D-02: Only strip LAST corner if it matches FIRST. Do NOT iterate for consecutive duplicates.
    - The validator runs AFTER at_least_three_corners. If dedup reduces below 3, at_least_three_corners already passed but the polygon is now invalid. To handle this: reorder so strip_close_polygon_duplicate runs FIRST (before at_least_three_corners), OR add a post-dedup check. Best approach: define strip_close_polygon_duplicate BEFORE at_least_three_corners so Pydantic runs dedup first, then the count check catches the edge case.

    IMPORTANT ordering: Place `strip_close_polygon_duplicate` ABOVE `at_least_three_corners` in the class body so it runs first. This way, if dedup drops from 3 to 2, the at_least_three_corners validator correctly rejects.
