    Public function `cluster_vertices`:

    ```python
    def cluster_vertices(
        polygons: dict[int, np.ndarray],
        planes: dict[int, Plane],
        tol: float = 1.0,
    ) -> tuple[dict[int, list[int]], list[tuple[int, int, np.ndarray]]]:
        """Cluster polygon vertices using three-pass expanding tolerance.

        Each pass uses a fraction of the base tolerance (0.3t, 0.6t, t).
        Cumulative: each pass adds unions on top of previous pass state.
        Uses scipy.cluster.hierarchy.DisjointSet for union-find.

        Returns (groups, items) where:
        - groups: dict mapping root index -> list of member indices into items
        - items: list of (panel_id, vertex_index, xyz_array) tuples
        """
    ```

    Implementation steps:
    1. Flatten all vertices: iterate `sorted(polygons.keys())` for determinism. For each pid, for each vertex index vi, append `(pid, vi, polygons[pid][vi])` to `items` list.
    2. Create `DisjointSet(range(len(items)))`.
    3. Pre-compute all pairwise squared distances (or use the O(N^2) nested loop like snapping.py).
    4. Three-pass loop: for each fraction in `_PASS_FRACTIONS`:
       - `pass_tol = tol * fraction`
       - `tol2 = pass_tol * pass_tol`
       - For each pair `(i, j)` where `i < j`:
         - If `float((items[i][2] - items[j][2]) @ (items[i][2] - items[j][2])) <= tol2`:
           - `ds.merge(i, j)` (DisjointSet uses `merge` not `union`)
       - Log: `log.info("clustering pass %.1ft (tol=%.3f m): %d items", fraction, pass_tol, len(items))`
    5. Build groups dict: for each index i in range(len(items)), find root via `ds[i]` (DisjointSet subscript returns the root). Group by root.
    6. Filter to groups with >0 members (keep all groups, including singletons, since graph.py needs them for valence counting).
    7. Log summary: `log.info("clustered %d vertices into %d groups (tol=%.3f m)", len(items), len(groups), tol)`
    8. Return `(groups, items)`.

    Key differences from snapping.py pattern:
    - Three passes instead of one (per TOPO-02)
    - `scipy.cluster.hierarchy.DisjointSet` instead of manual find/union (per STATE.md decision)
    - Returns `(groups, items)` instead of modified polygons (graph.py builds the graph from groups)
    - Does NOT write centroids back to polygons (that is the solver's job in Phase 2)
