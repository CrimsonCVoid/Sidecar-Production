# Phase 5 — UI Design Contract

> Visual and interaction contract for the labeling dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialize before Phase 5 execution) |
| Preset | User must run `npx shadcn@latest init` -- recommend neutral/zinc base, CSS variables mode |
| Component library | Radix (via shadcn) |
| Icon library | lucide-react (bundled with shadcn) |
| Font | Inter via `next/font/google` at weights 400, 600 |

**Rationale:** Phase 5 has minimal chrome (toolbar + header), but Phase 6 requires data tables, filter chips, badges, and a diff viewer. Initializing shadcn now establishes tokens and patterns that carry forward. The canvas itself uses Konva -- not shadcn components.

**shadcn components needed for Phase 5:**
- `button` -- toolbar actions (Save, Snap Preview, Undo, Redo, Delete Panel)
- `tooltip` -- valence dot panel-count tooltips, toolbar button labels
- `separator` -- toolbar section dividers
- `badge` -- panel count indicator, snap status
- `toast` -- save confirmation, error notifications
- `alert-dialog` -- destructive action confirmation (delete polygon)
- `dropdown-menu` -- canvas zoom/mode selector (if needed)

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within toolbar buttons |
| sm | 8px | Toolbar button internal padding, badge padding |

## Color

### Chrome (toolbar, header, background)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `zinc-950` / `#09090b` | Page background behind canvas -- dark to maximize hillshade contrast |
| Secondary (30%) | `zinc-900` / `#18181b` | Toolbar background, header background |
| Accent (10%) | `blue-500` / `#3b82f6` | Primary CTA button (Save Labels), active tool indicator, drawing mode cursor ring |
| Destructive | `red-500` / `#ef4444` | Delete polygon button, error toasts only |

**Dark background rationale:** The hillshade DSM image is a grayscale elevation raster. Dark chrome maximizes contrast with the hillshade content and reduces eye strain during extended labeling sessions. This is standard practice for image annotation tools (Labelbox, CVAT, Roboflow all use dark chrome).

Accent reserved for:
- "Save Labels" primary CTA button fill
- Active drawing tool highlight in toolbar
- Currently-drawing polygon stroke color
- Selected polygon highlight stroke

- Header: fixed, 48px height, full width. Back button (left), page title (center-left), Save Labels CTA (right).
- Toolbar: fixed below header, 44px height, full width. Tool buttons grouped with separators. Panel count badge on right.

- No sidebar in Phase 5. Panel list/properties are Phase 6 scope.

### Canvas Interaction Zones

| Zone | Area | Behavior |
|------|------|----------|
| Hillshade image | Entire canvas | Background raster, pannable and zoomable |
| Completed polygon | Polygon fill area | Click to select in Select mode. Shows panel ID label at centroid. |
| Vertex handle | 6px radius circle at each vertex | Visible on hover/select. Not draggable in Phase 5 (drag is out of scope per REQUIREMENTS.md). |
| First vertex zone | 10px radius around first vertex of active polygon | Shows green auto-close indicator. Click to close polygon. |
| Magnet zone | 12px radius around any existing vertex from another panel | Shows yellow snap indicator. Click snaps to existing vertex coords. Shift+click bypasses. |

## Interaction Contract

### Drawing Mode

1. User clicks canvas to place vertex (small white circle appears at click position).
2. A polyline connects placed vertices as visual feedback.
3. If click is within 12px of an existing vertex from another panel AND Shift is NOT held: vertex snaps to that position. Yellow ring indicator shown before click.
4. If Shift IS held: no snap, vertex placed at exact cursor position. No indicator shown.
5. When cursor moves within 10px of the first vertex: green ring appears around first vertex. Click closes the polygon.
6. On close: polygon fill + stroke applied from panel palette. Panel ID label placed at centroid. Panel added to Zustand store.
7. Minimum 3 vertices required. If user closes with fewer than 3 vertices, polygon is discarded (no error toast, just silent discard).

### Select Mode

1. Click on a completed polygon to select it. Selected polygon gets accent-colored stroke.
2. Click on empty canvas to deselect.
3. Press Delete key or click Delete toolbar button to delete selected polygon. Alert dialog confirmation shown first.
4. No vertex dragging in Phase 5 (explicitly out of scope).

### Undo / Redo

1. Cmd+Z undoes the last action. Actions: vertex placement, polygon completion, polygon deletion.
2. Cmd+Shift+Z redoes the last undone action.
3. Undo/redo history is per-session (clears on page navigation).
4. Undo survives polygon boundary crossings (e.g., undo after completing polygon A and starting polygon B restores the pre-completion state of polygon B, not polygon A).
5. Implementation: Zustand temporal middleware (zundo). State shape: `{ panels: PanelCorners[], activeDrawing: number[][] | null }`.

### Snap Preview

1. Click "Snap Preview" toolbar button.
2. Calls `POST /api/snap/preview` with current panel data in PanelsInput format.
3. On success: overlay valence-colored dots on canvas at each feature point position. Dots are Konva circles with color/radius per valence table above.
4. Hovering a valence dot shows tooltip: "{valence} panels meet here".
5. Snap preview overlay persists until user clicks "Snap Preview" again (toggles off) or modifies any polygon.
6. On error: toast with error message.

### Save

1. Click "Save Labels" button (primary CTA, always visible in header).
2. Calls `POST /api/labels/{sampleId}` with current panel data.
3. On success: toast "Labels saved ({N} panels)".
4. On error: toast "Save failed. Check your connection and try again."
5. No auto-save. User explicitly saves.

### Keyboard Shortcuts

| Shortcut | Action | Context |
|----------|--------|---------|
| Cmd+Z | Undo | Always |
| Cmd+Shift+Z | Redo | Always |
| Escape | Cancel active drawing / deselect | Drawing or Select mode |
| Delete / Backspace | Delete selected panel | Select mode, panel selected |
| D | Switch to Draw mode | Always |
| S | Switch to Select mode | Always |

## State Contract

### Zustand Store Shape (TypeScript)

```typescript
interface LabelerState {
  // Panel data
  panels: PanelData[];
  activeDrawing: number[][] | null; // vertices-in-progress, null when not drawing

  // UI state
  mode: "draw" | "select";
  selectedPanelIndex: number | null;
  snapPreview: SnapPreviewData | null;
  isSaving: boolean;
  isLoadingPreview: boolean;

  // Actions
  addVertex: (x: number, y: number) => void;
  closePolygon: () => void;
  deletePanel: (index: number) => void;
  setMode: (mode: "draw" | "select") => void;
  selectPanel: (index: number | null) => void;
  setSnapPreview: (data: SnapPreviewData | null) => void;
  loadPanels: (panels: PanelData[]) => void;
  cancelDrawing: () => void;
}

interface PanelData {
  id: number;
  corners_pix: number[][]; // [[col, row], ...] matching PanelCorners schema
}

interface SnapPreviewData {
  features: FeatureNode[];
  snappedPolygons: Record<string, number[][]>;
}

interface FeatureNode {
  id: number;
  valence: number;
  position_xyz: number[] | null;
  panel_ids: number[];
}
```

Temporal middleware (zundo) wraps `panels` and `activeDrawing` for undo/redo. UI state (`mode`, `selectedPanelIndex`, `snapPreview`, `isSaving`, `isLoadingPreview`) is excluded from undo history.

## Zod Schemas (API boundary validation)

```typescript
// Mirror of Python PanelCorners / PanelsInput
const PanelCornersSchema = z.object({
  id: z.number().int(),
  corners_pix: z.array(z.array(z.number()).length(2)).min(3),
});

const PanelsInputSchema = z.object({
  panels: z.array(PanelCornersSchema),
  res_m: z.number().nullable().optional(),
  shape: z.array(z.number().int()).nullable().optional(),
  panel_count: z.number().int().nullable().optional(),
  panel_pixel_counts: z.record(z.string(), z.number().int()).nullable().optional(),
});

const FeatureNodeSchema = z.object({
  id: z.number().int(),
  valence: z.number().int(),
  position_xyz: z.array(z.number()).length(3).nullable(),
  panel_ids: z.array(z.number().int()),
});

const SnapPreviewResponseSchema = z.object({
  feature_graph: z.object({
    features: z.array(FeatureNodeSchema),
    edges: z.array(z.object({
      panel_a: z.number().int(),
      panel_b: z.number().int(),
      feature_ids: z.array(z.number().int()),
    })),
  }),
  snapped_polygons: z.record(z.string(), z.array(z.array(z.number()))),
});

const LabelDataSchema = z.object({
  sample_id: z.string(),
  panels: z.array(PanelCornersSchema),
});
```

These schemas MUST mirror the Pydantic models in `roof_pipeline/api/schemas.py` and `roof_pipeline/panel_snap_v2/schema.py`. Pitfall #5 from STATE.md: schema drift between Pydantic and Zod.

## Error Capture Contract (OBSERVABILITY-01b)

### Browser-side error capture

| Error Type | Capture Method | Destination |
|------------|----------------|-------------|
| Unhandled exceptions | `window.addEventListener("error", ...)` | POST `/api/errors` on FastAPI sidecar |
| Unhandled promise rejections | `window.addEventListener("unhandledrejection", ...)` | POST `/api/errors` on FastAPI sidecar |
| API call failures | try/catch in fetch wrapper | Toast notification + POST `/api/errors` |
| Konva rendering errors | Konva error event handler | POST `/api/errors` |

Error payload shape:
```typescript
interface BrowserError {
  timestamp: string;       // ISO 8601
  page: string;            // e.g., "/labeling/abc123"
  error_type: string;      // "unhandled_exception" | "unhandled_rejection" | "api_error" | "render_error"
  message: string;
  stack: string | null;
  user_agent: string;
  sample_id: string | null;
}
```

Backend endpoint: `POST /api/errors` on FastAPI sidecar. Logs the error as a structured JSON line using the existing logging middleware. No Sentry in Phase 5 (single-user MVP).

## Accessibility Baseline

| Concern | Contract |
|---------|----------|
| Keyboard navigation | All toolbar buttons reachable via Tab. Enter/Space activates. Shortcuts documented in tooltips. |
| Color redundancy | Valence dots use size + color (not color alone). Polygon selection uses stroke weight + color. |
| Focus indicators | shadcn default focus rings on all interactive elements. |
| Canvas accessibility | Canvas is a visual tool with no screen-reader alternative in Phase 5. Toolbar buttons have aria-labels. |
| Touch targets | Minimum 44px height on all toolbar buttons. |
| Contrast | Dark chrome (zinc-950/900) with white/light text. Minimum 4.5:1 contrast ratio for body text. |
