def _z_on_plane(x: float, y: float, plane: Plane) -> float:
    """Z-coordinate on ``plane`` at world (x, y). Uses n . p = d."""
    nx, ny, nz = plane.normal
    if abs(nz) < 1e-9:
        return float(plane.centroid[2])
    return (plane.d - nx * x - ny * y) / nz

**Module structure** (follow clustering.py pattern):

```python
"""Valence-aware apex solver: resolve clustered vertices to geometrically correct positions.

Dispatches by cluster valence:
  valence-1: unshared vertex, no action needed
  valence-2: XY centroid + per-plane Z reconstruction (D-02, TOPO-05)
  valence-3: closed-form 3x3 plane intersection via numpy.linalg.solve (TOPO-06)
  valence-4+: weighted least-squares via numpy.linalg.lstsq (TOPO-07)

Condition-number guards (D-01):
  cond > 1e8: fall back to centroid with WARNING
  cond > 1e12: hard-fail with RuntimeError
"""

from __future__ import annotations

import logging

import numpy as np

from ..planes import Plane
from .clustering import cluster_vertices
from .graph import build_feature_graph

log = logging.getLogger(__name__)

# ---------------------------------------------------------------------------
# Constants
# ---------------------------------------------------------------------------

_COND_WARN = 1e8
_COND_FAIL = 1e12
```

**Private helpers to implement:**

1. `_z_on_plane(x, y, plane)` -- copy from snapping.py lines 34-43 (do NOT import from snapping -- snapping.py is being superseded). Exact implementation:
```python
def _z_on_plane(x: float, y: float, plane: Plane) -> float:
    nx, ny, nz = plane.normal
    if abs(nz) < 1e-9:
        return float(plane.centroid[2])
    return (plane.d - nx * x - ny * y) / nz
```

2. `_solve_valence2(members, items, planes)` -- per D-02:
   - Compute XY centroid of all member vertices: `xy_mean = mean of items[m][2][:2] for m in members`
   - For each member m: get panel_id = items[m][0], vi = items[m][1]. Reconstruct Z via `_z_on_plane(xy_mean[0], xy_mean[1], planes[panel_id])`. Set result[m] = [xy_mean[0], xy_mean[1], z].
   - Return dict mapping member_index -> solved_xyz.

3. `_solve_valence3(members, items, planes)` -- per TOPO-06:
   - Collect the 3 distinct panel IDs from members.
   - Build the 3x3 normal matrix N where row i = planes[pid_i].normal.
   - Build RHS d where d[i] = planes[pid_i].d.
   - Check condition number: `cond = np.linalg.cond(N)`.
   - If `cond > _COND_FAIL`: raise RuntimeError with format per D-01.
   - If `cond > _COND_WARN`: log WARNING per D-01 format, fall back to `_solve_valence2`.
   - Else: `apex = np.linalg.solve(N, d)`. All members get this apex point.
   - Return dict mapping member_index -> solved_xyz.

4. `_solve_valence4plus(members, items, planes)` -- per TOPO-07:
   - Collect distinct panel IDs.
   - Build Nx3 normal matrix (one row per unique panel).
   - Build weight diagonal: `w[i] = 1.0 / planes[pid_i].rms_residual`.
   - Apply weights: `N_w = diag(w) @ N`, `d_w = diag(w) @ d`.
   - Check condition number of N_w.
   - If `cond > _COND_FAIL`: raise RuntimeError per D-01.
   - If `cond > _COND_WARN`: log WARNING per D-01, fall back to `_solve_valence2`.
   - Else: `apex, _, _, _ = np.linalg.lstsq(N_w, d_w, rcond=None)`. All members get this apex.
   - Return dict mapping member_index -> solved_xyz.

**Public function:**

```python
def solve_apices(
    polygons: dict[int, np.ndarray],
    planes: dict[int, Plane],
    graph: dict,
    tol: float = 1.0,
) -> dict[int, np.ndarray]:
```

Algorithm:
1. Copy-on-write: `out = {pid: poly.copy() for pid, poly in polygons.items()}`
2. Call `cluster_vertices(out, planes, tol=tol)` to get `(groups, items)`.
3. For each cluster (root, members) in groups:
   - Get panel_ids = sorted unique panel IDs from members.
   - valence = len(panel_ids).
   - If valence == 1: skip (unshared vertex).
   - If valence == 2: call `_solve_valence2`.
   - If valence == 3: call `_solve_valence3`.
   - If valence >= 4: call `_solve_valence4plus`.
4. Write solved positions back (TOPO-08): for each solved member m, get (pid, vi, _) from items[m], set `out[pid][vi] = solved_xyz`.
5. Log summary per D-03: `log.info("snap_v2: solved %d apices (%d ridge, %d hip), %d corner snaps, %d fallbacks", ...)`.
6. Return `out`.

Track counters for the summary log: ridge_count (valence-3 solved without fallback), hip_count (valence-4+ solved without fallback), corner_count (valence-2), fallback_count.
