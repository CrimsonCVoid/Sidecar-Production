## File Format Dependencies

### Inputs

**GeoTIFF DSM (Digital Surface Model):**
- Format: GeoTIFF (`.tif`)
- Reader: `rasterio.open()` in `roof_pipeline/run_real.py` and `roof_pipeline/label_panels.py`
- Content: 2D float array of elevations in meters
- Metadata used: pixel resolution (`src.transform.a`), nodata value (`src.nodata`)
- Source: External (e.g., Google Solar API tiles, drone photogrammetry)

**Panel Mask:**
- Format: NumPy binary (`.npy`)
- Reader: `np.load()` in `roof_pipeline/run_real.py`
- Content: 2D uint8 array, 0 = background, 1..N = panel IDs
- Generated by: `roof_pipeline/label_panels.py` interactive tool

**Panel Click Coordinates (sidecar):**
- Format: JSON (`.json`, same stem as mask `.npy`)
- Reader: `json.load()` in `roof_pipeline/boundaries.py` (`polygons_from_clicks()`)
- Content: `{ "panels": [{ "id": int, "corners_pix": [[col, row], ...] }], "res_m": float, ... }`
- Generated by: `roof_pipeline/label_panels.py` (saved alongside mask)

### Outputs

**3D Mesh - OBJ:**
- Format: Wavefront OBJ (`.obj`)
- Writer: `trimesh.Trimesh.export()` in `roof_pipeline/mesh.py`
- Path: `output/roof.obj`

**3D Mesh - glTF:**
- Format: glTF 2.0 (`.gltf`)
- Writer: `trimesh.Trimesh.export()` in `roof_pipeline/mesh.py` (uses `pygltflib` internally)
- Path: `output/roof.gltf`

**Cut Sheet PDF:**
- Format: PDF (multi-page, letter size)
- Writer: ReportLab `SimpleDocTemplate` in `roof_pipeline/cutsheets.py`
- Content: Cover page with plan view + panel index table; one page per panel with dimensioned true-length drawing, 3D context inset, area/slope/azimuth metrics
- Path: `output/cutsheets.pdf`

**TS Exporter JSON:**
- Format: JSON (custom schema, `schema_version: 1`)
- Writer: `json.dump()` in `roof_pipeline/ts_export.py`
- Purpose: Intermediate format mirroring the TypeScript `PDF_Exporter` `DrawType`/`Draw` data model, consumed by both the Python validation renderer and the browser-side TS class
- Content: `draw_types` array with `panel_outline`, `edge_label`, `angle_label`, `panel_header`, `plan_outline`, `plan_label`, `cover_title` types; each contains `draws` with polyline points in inches + text labels
- Coordinate convention: `{ x: -v_inches, z: u_inches }` mapped via `pageX = z*scale+300, pageY = -x*scale+400`
- Path: `output/cutsheets.ts.json`

**TS-Render Validation PDF:**
- Format: PDF (600x800 pt pages)
- Writer: ReportLab `Canvas` in `roof_pipeline/ts_render_pdf.py`
- Purpose: Python-side preview of what the browser TS PDF renderer will produce; validates the JSON schema
- Path: `output/cutsheets.ts.pdf`

**Shop Drawings PDF:**
- Format: PDF (mixed page sizes: ANSI B portrait/landscape + Letter)
- Writer: ReportLab `Canvas` in `roof_pipeline/shop_drawings.py`
- Content: 4-page fabricator-facing document (panel layout plan, edge/trim diagram, sheet cut list, combined edge detail)
- Path: `output_real/shop_drawings.pdf`

## Third-Party Library Integrations

**NumPy (`numpy`):**
- Role: Core numerical backbone
- Used for: SVD plane fitting (`np.linalg.svd`), coordinate transforms, array math, polygon area (shoelace), bilinear sampling
- Files: All 13 modules

**SciPy (`scipy`):**
- Role: Image morphology for building detection
- Used for: `binary_closing`, `binary_opening`, `label` from `scipy.ndimage`
- Files: `roof_pipeline/label_panels.py`

**OpenCV (`cv2`):**
- Role: Contour extraction and simplification
- Used for: `cv2.findContours()` (external contour of binary mask), `cv2.approxPolyDP()` (Ramer-Douglas-Peucker simplification), `cv2.contourArea()` (largest contour selection)
- Files: `roof_pipeline/boundaries.py`

**scikit-image (`skimage`):**
- Role: Polygon rasterization
- Used for: `skimage.draw.polygon()` to convert clicked corners into a filled pixel mask
- Files: `roof_pipeline/label_panels.py`

**trimesh:**
- Role: 3D mesh construction and export
- Used for: `trimesh.Trimesh()` construction, `trimesh.util.concatenate()` for multi-panel merge, `.export()` for OBJ and glTF output
- Files: `roof_pipeline/mesh.py`, `roof_pipeline/cutsheets.py` (reads mesh for 3D inset rendering)

**mapbox-earcut:**
- Role: Polygon triangulation
- Used for: `earcut.triangulate_float64()` to triangulate planar polygons in local 2D frame
- Files: `roof_pipeline/mesh.py`

**rasterio:**
- Role: GeoTIFF I/O
- Used for: `rasterio.open()` to read DSM elevation rasters, extract pixel resolution and nodata values
- Files: `roof_pipeline/run_real.py`, `roof_pipeline/label_panels.py`

**matplotlib:**
- Role: Visualization and interactive GUI
- Used for: Dimensioned panel drawings (PNG), 3D inset views (Poly3DCollection), plan view rendering, hillshade terrain visualization, interactive labeler GUI (click/scroll/pan events)
- Backend: `Agg` (headless) in `cutsheets.py`; default GUI backend in `label_panels.py`
- Files: `roof_pipeline/cutsheets.py`, `roof_pipeline/label_panels.py`

**ReportLab:**
- Role: PDF document generation
- Used for: `SimpleDocTemplate` with flowables (Image, Table, Paragraph, PageBreak) in cutsheets; raw `Canvas` for TS-render validation and shop drawings
- Files: `roof_pipeline/cutsheets.py`, `roof_pipeline/ts_render_pdf.py`, `roof_pipeline/shop_drawings.py`

**pygltflib:**
- Role: glTF format support
- Used indirectly by trimesh for `.gltf` export
- Files: Not directly imported (transitive via trimesh)
